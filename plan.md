# PhotoView - Windows 图片查看器 & 合集管理工具

## 一、项目背景

下载了大量图片，按合集（文件夹）分类组织，但缺乏高效的管理和浏览手段。Windows 自带图片查看器功能有限，不支持 webp，且无法对合集进行增删改查管理。需要一个本地化的图片查看 + 合集管理一体化工具。

---

## 二、核心痛点分析

### 2.1 浏览体验痛点

| # | 痛点 | 说明 |
|---|------|------|
| 1 | **格式支持不全** | Windows 默认查看器不支持 webp、avif 等新格式，需额外安装编解码器 |
| 2 | **大图加载慢** | 高分辨率图片（4K+）打开慢，缩放卡顿 |
| 3 | **切换不流畅** | 文件夹内图片多时，上一张/下一张切换有延迟 |
| 4 | **缩略图生成慢** | 资源管理器缩略图缓存机制差，大文件夹加载缩略图极慢 |
| 5 | **无快速预览** | 无法在不打开图片的情况下快速浏览内容 |

### 2.2 合集管理痛点

| # | 痛点 | 说明 |
|---|------|------|
| 1 | **合集信息缺失** | 文件夹只有名称，无封面、描述、标签、评分等元数据 |
| 2 | **查找困难** | 合集多了以后，只能靠文件夹名搜索，无法按标签/评分/日期等多维度检索 |
| 3 | **合集操作繁琐** | 合并、拆分、移动合集需要手动操作文件系统，容易出错 |
| 4 | **重复图片** | 同一张图片可能存在于多个合集中，占用空间且难以发现 |
| 5 | **合集状态不清** | 不知道哪些合集已看/未看/收藏，缺乏状态标记 |
| 6 | **排序方式单一** | 只能按文件系统排序，无法按评分/图片数/创建时间等自定义排序 |

### 2.3 图片管理痛点

| # | 痛点 | 说明 |
|---|------|------|
| 1 | **无法标记** | 无法对单张图片进行收藏、评分、标记 |
| 2 | **无EXIF查看** | 无法快速查看图片的拍摄信息、分辨率、文件大小等 |
| 3 | **批量操作难** | 无法批量移动、删除、重命名图片 |
| 4 | **跨合集查找** | 无法在所有合集中搜索某类图片 |

## 三、需要解决的关键问题

### 3.1 图片解码与渲染

- **多格式支持**：jpg/jpeg, png, gif, bmp, ico, svg, webp, avif, tiff
- **高效解码**：大图异步加载 + 渐进式渲染，避免 UI 卡顿
- **缩略图系统**：后台预生成缩略图 + 持久化缓存，避免重复解码
- **内存管理**：LRU 缓存策略，控制内存占用，支持万级图片浏览不 OOM

### 3.2 合集数据模型

- **元数据存储**：合集名称、封面、描述、标签、评分、创建时间、图片数量、总大小
- **持久化方案**：本地 SQLite 数据库，轻量无需服务端
- **文件夹同步**：监听文件系统变化，自动检测新增/删除的图片和文件夹
- **无外键约束**：所有表关联通过应用层逻辑维护（项目架构约束）

### 3.3 合集 CRUD

- **创建**：导入文件夹作为合集，自动扫描图片、生成缩略图、提取元数据
- **读取**：列表视图 + 网格视图，支持多维度排序和筛选
- **更新**：修改合集信息（名称/描述/标签/评分/封面），移动图片到其他合集
- **删除**：删除合集记录（可选是否删除物理文件）

### 3.4 图片检索与过滤

- **全文搜索**：按文件名、合集名、标签搜索
- **多维过滤**：按格式/分辨率区间/文件大小区间/评分/日期范围过滤
- **标签系统**：为合集和图片打标签，支持标签的增删改查
- **收藏夹**：跨合集的收藏图片快速访问

### 3.5 重复检测

- **感知哈希**：pHash/dHash 算法检测视觉相似的图片
- **文件哈希**：MD5/SHA256 检测完全相同的文件
- **去重建议**：展示重复组，提供保留/删除建议

## 四、技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| **框架** | Tauri 2 (Rust + WebView) | 轻量、高性能、原生文件系统访问、小安装包 |
| **前端** | React + TypeScript + TailwindCSS | 生态成熟，组件丰富，开发效率高 |
| **UI组件** | shadcn/ui | 现代化设计，高度可定制 |
| **图片解码** | Rust image crate + webp/avif 解码库 | 原生性能，支持多线程并行解码 |
| **数据库** | SQLite (via rusqlite) | 轻量、零配置、单文件、事务支持 |
| **缩略图缓存** | 本地文件系统 + SQLite 索引 | 缩略图存文件，路径存数据库 |
| **文件监听** | notify crate | 跨平台文件系统事件监听 |
| **构建打包** | Tauri CLI | 一键打包 Windows 安装程序 |

## 六、数据库设计（无外键约束）

```sql
-- 合集表
CREATE TABLE collections (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,
    description TEXT DEFAULT '',
    cover_path  TEXT,                -- 封面图片路径
    folder_path TEXT NOT NULL UNIQUE,-- 源文件夹路径
    rating      INTEGER DEFAULT 0,   -- 0-5 评分
    is_favorite INTEGER DEFAULT 0,   -- 0否 1是
    view_count  INTEGER DEFAULT 0,
    last_viewed TEXT,                -- ISO8601 时间戳
    created_at  TEXT DEFAULT (datetime('now')),
    updated_at  TEXT DEFAULT (datetime('now'))
);

-- 图片表
CREATE TABLE photos (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    collection_id  INTEGER NOT NULL,  -- 应用层维护关联
    file_name      TEXT NOT NULL,
    file_path      TEXT NOT NULL UNIQUE,
    file_size      INTEGER DEFAULT 0,
    width          INTEGER DEFAULT 0,
    height         INTEGER DEFAULT 0,
    format         TEXT,              -- jpg/png/webp/...
    file_hash      TEXT,              -- SHA256，用于去重
    perceptual_hash TEXT,             -- 感知哈希，用于相似检测
    rating         INTEGER DEFAULT 0,
    is_favorite    INTEGER DEFAULT 0,
    exif_data      TEXT,              -- JSON 格式存储 EXIF
    created_at     TEXT DEFAULT (datetime('now')),
    updated_at     TEXT DEFAULT (datetime('now'))
);

-- 标签表
CREATE TABLE tags (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    name  TEXT NOT NULL UNIQUE,
    color TEXT DEFAULT '#6366f1'      -- 标签颜色
);

-- 合集-标签关联表
CREATE TABLE collection_tags (
    collection_id INTEGER NOT NULL,
    tag_id        INTEGER NOT NULL,
    PRIMARY KEY (collection_id, tag_id)
);

-- 图片-标签关联表
CREATE TABLE photo_tags (
    photo_id INTEGER NOT NULL,
    tag_id   INTEGER NOT NULL,
    PRIMARY KEY (photo_id, tag_id)
);

-- 缩略图缓存表
CREATE TABLE thumbnail_cache (
    photo_id    INTEGER PRIMARY KEY,
    thumb_path  TEXT NOT NULL,        -- 缩略图文件路径
    thumb_size  INTEGER DEFAULT 0,    -- 缩略图文件大小
    width       INTEGER DEFAULT 0,
    height      INTEGER DEFAULT 0,
    generated_at TEXT DEFAULT (datetime('now'))
);

-- 导入历史表（避免重复导入）
CREATE TABLE import_history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    folder_path TEXT NOT NULL UNIQUE,
    imported_at TEXT DEFAULT (datetime('now'))
);
```

**应用层关联约束**：
- 删除合集时，应用层负责先删除 `collection_tags`、`photos`（及关联的 `photo_tags`、`thumbnail_cache`）
- 删除图片时，应用层负责先删除 `photo_tags`、`thumbnail_cache`
- 所有写操作在应用层事务中执行，保证数据一致性

## 十、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| webp/avif 解码性能差 | 大图加载慢 | 多线程预解码 + 渐进式渲染 |
| 大量缩略图占用磁盘 | 磁盘空间不足 | 可配置缓存上限 + LRU 清理 |
| 文件系统监听资源消耗 | 后台持续占用 | 可配置监听范围，仅监听已导入目录 |
| SQLite 并发写入 | 数据损坏 | WAL 模式 + 应用层写队列 |
| 图片文件被外部修改/删除 | 数据不一致 | 启动时校验 + 定期一致性检查 |
